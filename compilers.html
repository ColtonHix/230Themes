The big twist

Perl is mainly an interpreted language!

Perl is normally an interpreted language, although Perl 6 “Raku” is not. Perl binaries are usually already installed on most Linux/UNIX like system, it even exists on our web directory and onyx! Run perl -v to check the version installed. ActivePerl is one of the recommended methods of installation for any platform. It provides binaries for Perl and a set of modules. You can view the official installation instructions here.

Perl does not have it’s own compiler, but Strawberry perl uses gcc. ActivePerl exists for Windows/Mac/Linux, while Microsoft also has Strawberry Perl “A 100% Open Source Perl for Windows that is exactly the same as Perl everywhere else; this includes using modules from CPAN, without the need for binary packages.”
The Perl interpreter can be installed on web servers (Apache) and database servers!

Mac OS X

Binaries

Mac OS X should already have Perl installed, run perl -v in the terminal to check the version installed. Binaries can still be downloaded from ActivePerl.

ActivePerl Downloads
  1. Use the Mac OS X Editors and Terminals
  2. Install "Command Line Tools for Xcode", either directly or through Xcode, both available from Apple Developer downloads (free registration required). Xcode can also be installed through the App Store application.
  3. Open the Terminal.app (found in Applications -> Utilities) and copy & paste the command below into it (then press the 'return' key):
    curl -L http://xrl.us/installperlosx | bash
  4. Once this is finished (it takes several minutes), quit the Terminal app.

Linux/Unix

Binaries

Linux/Unix systems may already have Perl installed, run perl -v in the terminal to check the version installed. Binaries can still be downloaded from ActivePerl.

ActivePerl Downloads
  1. For the below installation, you must have a compiler such as gcc already installed
  2. Open the terminal and copy & paste the command below into it (then press the 'return' key):
    curl -L http://xrl.us/installperlnix | bash
